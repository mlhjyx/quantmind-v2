#### Task 1: 市场冲击成本集成SimBroker + 重跑基线（P0）

**角色**: arch(编码) + quant(审交易成本建模)
**交叉审查**: qa验测试, quant审统计合理性
**必读设计文档**: DEV_BACKTEST_ENGINE.md §4.5(SlippageModel), DEV_PARAM_CONFIG.md §3.7+§5.3, DESIGN_V5 §6

**现状**:
- `slippage_model.py` 已实现平方根冲击模型（单一impact_coeff=0.1）
- SimBroker还在用固定bps（`BacktestConfig.slippage_bps=10.0`）
- DEV_BACKTEST_ENGINE.md设计了更完善的`SlippageConfig`（市值分层k系数: k_large=0.05, k_mid=0.10, k_small=0.15）
- DEV_PARAM_CONFIG.md §3.7定义了滑点参数为L2级可配置（输入框+范围约束+验证规则）
- `param_defaults.py`目前只有`backtest.slippage_bps`，缺少市值分层参数

**实现方案**:

1. **升级SlippageConfig**（按设计文档）:
   ```python
   @dataclass(frozen=True)
   class SlippageConfig:
       k_large: float = 0.05    # 500亿+ 大盘
       k_mid: float = 0.10      # 100-500亿 中盘
       k_small: float = 0.15    # 100亿以下 小盘
       sell_penalty: float = 1.2  # 卖出惩罚
       base_bps: float = 5.0     # 基础滑点(bid-ask spread)
   ```

2. **参数可配置性**（按DEV_PARAM_CONFIG.md §3.7+§5.3）:
   - `param_defaults.py`新增以下参数定义（保留原有`backtest.slippage_bps`用于fixed模式，新增市值分层参数用于volume_impact模式）：
     ```python
     # 滑点模型参数（L2级，用户可通过前端或API编辑）
     ParamDef(key="backtest.slippage_mode", default_value="volume_impact",
              param_type=ParamType.ENUM, module=ParamModule.BACKTEST,
              description="滑点模型: volume_impact(动态) / fixed(固定)",
              allowed_values=["volume_impact", "fixed"])
     ParamDef(key="backtest.slippage_k_large", default_value=0.05,
              param_type=ParamType.FLOAT, module=ParamModule.BACKTEST,
              description="滑点系数-大盘(500亿+)", min_value=0.01, max_value=0.3)
     ParamDef(key="backtest.slippage_k_mid", default_value=0.10,
              param_type=ParamType.FLOAT, module=ParamModule.BACKTEST,
              description="滑点系数-中盘(100-500亿)", min_value=0.01, max_value=0.3)
     ParamDef(key="backtest.slippage_k_small", default_value=0.15,
              param_type=ParamType.FLOAT, module=ParamModule.BACKTEST,
              description="滑点系数-小盘(100亿以下)", min_value=0.01, max_value=0.3)
     ParamDef(key="backtest.slippage_base_bps", default_value=5.0,
              param_type=ParamType.FLOAT, module=ParamModule.BACKTEST,
              description="基础滑点(bps, bid-ask spread)", min_value=0.0, max_value=20.0)
     ParamDef(key="backtest.volume_cap_pct", default_value=0.10,
              param_type=ParamType.FLOAT, module=ParamModule.BACKTEST,
              description="单日成交量上限(%)", min_value=0.05, max_value=0.30)
     ```
   - 参数约束验证（DEV_PARAM_CONFIG.md §5.3）:
     ```python
     ("slippage_k_large <= slippage_k_mid <= slippage_k_small",
      "滑点系数大盘≤中盘≤小盘")
     ```
   - SimBroker/BacktestConfig从ParamService读取这些参数（而不是硬编码）
   - API `/api/params/backtest`返回这些参数供前端展示和编辑

3. **修改slippage_model.py**:
   - `volume_impact_slippage()`的`impact_coeff`参数改为根据`market_cap`自动选择k系数
   - 保持现有函数签名的向后兼容（impact_coeff作为fallback）
   - 新增`SlippageConfig`参数重载

4. **集成到SimBroker**:
   - `BacktestConfig`新增`slippage_mode: str = 'volume_impact'`和`slippage_config: SlippageConfig`
   - SimBroker的`_calc_slippage()`方法改为：
     - `slippage_mode='volume_impact'`: 调用`volume_impact_slippage()`，需要传入当日`daily_amount`和`market_cap`
     - `slippage_mode='fixed'`: 保持原固定bps行为
   - SimBroker执行时从行情DataFrame获取`daily_amount`（日成交额）和`total_mv`（总市值）
   - 参数从ParamService/配置读取，不硬编码

5. **回测报告新增冲击成本分解**:
   - `PerformanceReport`新增`avg_impact_bps`和`impact_cost_breakdown`字段
   - 回测输出每笔交易的base_bps + impact_bps分解

6. **重跑基线**（关键步骤）:
   - 用volume-impact模型（默认参数）重跑2021-2025全期回测
   - 更新CLAUDE.md基线Sharpe和毕业标准阈值
   - 新旧基线对比表写入PROGRESS.md

**验收标准**:
- 同一v1.1配置跑两次回测（fixed vs volume_impact），Sharpe有差异

#### Task 2: gap_hours时间戳采集 + DB迁移（P0）

**角色**: arch
**交叉审查**: qa验测试
**必读设计文档**: DEV_SCHEDULER.md §P6(三阶段调度)

**现状**:
- `calc_signal_execution_gap_hours()`函数+8个测试已存在
- `PaperTradingService`已调用该函数
- 但PT脚本没有记录`signal_generated_at`时间戳到DB
- DDL中`signals`表和`trade_log`表没有该字段

**实现方案**:

1. **DB迁移**:
   ```sql
   -- signals表新增信号生成时间戳
   ALTER TABLE signals ADD COLUMN signal_generated_at TIMESTAMPTZ DEFAULT NULL;
   COMMENT ON COLUMN signals.signal_generated_at IS '信号生成时间（UTC），用于计算gap_hours毕业指标';

   -- trade_log表新增执行确认时间戳（如果没有的话）
   ALTER TABLE trade_log ADD COLUMN executed_at TIMESTAMPTZ DEFAULT NULL;
   COMMENT ON COLUMN trade_log.executed_at IS '实际执行确认时间（UTC）';
   ```
   - `ALTER TABLE ... ADD COLUMN ... DEFAULT NULL`不影响现有数据
   - 前3天数据的gap_hours无法回溯，毕业评估排除NULL记录

2. **PT信号阶段写入时间戳**:
   - `run_paper_trading.py`的Step 3（信号生成）完成后：
     ```python
     signal_generated_at = datetime.now(timezone.utc)
     # 写入signals表的signal_generated_at字段
     ```

3. **PT执行阶段写入时间戳**:
   - Step 5（执行确认）完成后：
     ```python
     executed_at = datetime.now(timezone.utc)
     # 写入trade_log表的executed_at字段
     ```

4. **PaperTradingService数据源对接**:
   - `_calc_execution_metrics()`从DB查询两个时间戳列表
   - 传给已有的`calc_signal_execution_gap_hours()`

**验收标准**:
- PT信号阶段后，`SELECT signal_generated_at FROM signals`有非NULL值
- PT执行阶段后，`SELECT executed_at FROM trade_log`有非NULL值
- `check_graduation.py`能输出gap_hours指标

#### Task 3: PT心跳watchdog（P0）

**角色**: TL直接做（轻量，<50行）
**交叉审查**: 无（轻量任务§1.2允许）

**现状**:
- PT信号阶段完成后发钉钉日报 ✅
- 健康预检失败发钉钉P0 ✅
- **但如果Task Scheduler本身没触发，没有任何机制检测**

**实现方案**:

1. **心跳记录**（在PT信号阶段末尾追加）:
   ```python
   # run_paper_trading.py Step 4之后
   heartbeat_file = Path("D:/quantmind-v2/logs/pt_heartbeat.json")
   heartbeat_file.write_text(json.dumps({
       "trade_date": str(trade_date),
       "completed_at": datetime.now().isoformat(),
       "phase": "signal",
       "status": "ok"
   }))
   ```

2. **watchdog脚本**（`scripts/pt_watchdog.py`，独立Task Scheduler任务20:00触发）:
   ```python
   def check_heartbeat():
       heartbeat_file = Path("D:/quantmind-v2/logs/pt_heartbeat.json")
       if not heartbeat_file.exists():
           send_alert("P0", "PT心跳缺失", "心跳文件不存在")
           return

       data = json.loads(heartbeat_file.read_text())
       last_date = date.fromisoformat(data["trade_date"])
       today = get_latest_trading_day()  # 最近交易日

       if last_date < today:
           send_alert("P0", "PT今天没跑",
                       f"最后心跳: {last_date}, 今天: {today}")
   ```

3. **Task Scheduler注册**: `schtasks`注册每日20:00执行watchdog

**验收标准**:
- 正常运行后heartbeat文件更新
- 手动删除heartbeat文件 → watchdog在20:00发P0钉钉告警
