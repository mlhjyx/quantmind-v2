## 2. Q11.2 SQL Injection (实测)

```bash
grep -rE "execute\(\s*f['\"]|execute\(.*\.format\(|execute\(.*\+.*\)" backend/app/ --include="*.py"
```

**实测 2 hits, 全在 `data_orchestrator.py`**:

| 路径:行 | 语句 | 用户输入风险? | D3-KEEP? |
|---|---|---|---|
| `backend/app/services/data_orchestrator.py:460` | `cur.execute(f"SELECT MAX(trade_date) FROM {tbl}")` | ❌ tbl 来自 `tables` 参数, 调用方传, default `["klines_daily","daily_basic","moneyflow_daily"]` hardcoded | D3-KEEP (无 user input 路径) |
| `backend/app/services/data_orchestrator.py:778` | `cur.execute(f"SELECT MAX(trade_date) FROM {asset}")` | ❌ asset_names 同样内部调用 | D3-KEEP |

### F-D3A-10 (P3) — Defense in depth

虽然当前无 user input 路径, 但 f-string SQL 没 sanitize 是 fragile pattern. **建议加 table allowlist** (validate `tbl in ALLOWED_TABLES` 后才拼接). 不是 imminent risk, 但 future-proof.

```python
# 修法建议 (批 2 P3 候选):
ALLOWED_TABLES = {"klines_daily", "daily_basic", "moneyflow_daily", ...}
if tbl not in ALLOWED_TABLES:
    raise ValueError(f"unsupported table: {tbl}")
cur.execute(f"SELECT MAX(trade_date) FROM {tbl}")
```
