### Phase G — ML联合优化 + 组合升级（最高优先级）

> **核心洞察**: RD-Agent(Q)用ML联合优化实现了2x收益提升，用70%更少的因子。
> QuantMind的5因子等权是线性组合天花板，需要让ML学因子交互。

#### G1. LightGBM Walk-Forward v1.3

| 属性 | 值 |
|------|------|
| **任务** | 实现ML_WALKFORWARD_DESIGN.md中905行的完整设计：7-fold Walk-Forward + Optuna超参搜索 + GPU训练 |
| **来源** | ML_WALKFORWARD_DESIGN.md（已完整设计）+ RD-Agent联合优化思路 |
| **核心改变** | 5因子等权（线性）→ LightGBM预测（非线性）。ML能捕捉因子交互（如高换手+低波动的组合效应） |
| **特征** | 51个（5基线+12多尺度+8资金流+10价格行为+8基本面+8市场状态）|
| **目标** | OOS Sharpe ≥ 0.8, paired bootstrap p < 0.05 vs 等权基线 |
| **上线标准** | OOS Sharpe ≥ 1.10 + p < 0.05 + 6红线全过 → 影子PT → v1.3升级 |
| **估时** | 10天（Optuna 5h + 7fold训练70min + 回测分析） |
| **GPU** | RTX 5070, device_type=gpu, max_bin=63 |

#### G2. 风险平价替代等权

| 属性 | 值 |
|------|------|
| **任务** | 等权分配→风险平价（Risk Parity）分配。高波股分少、低波股分多，总风险贡献均匀 |
| **理由** | 等权隐含假设15只股票风险相等，但小盘股波动率是大盘股2-3x。等权意味着小盘股贡献70%+组合风险 |
| **公式** | w_i ∝ 1/σ_i，归一化后 Σw_i = 1 - cash_buffer |
| **预期** | MDD改善20-30%（从-28%降到-20%以下） |
| **估时** | 3天 |
| **验证** | 5年回测对比：等权 vs 风险平价，MDD/Sharpe/年度分解 |

#### G3. 因子+模型联合迭代框架

| 属性 | 值 |
|------|------|
| **任务** | 参考RD-Agent(Q)：因子搜索和模型训练交替进行，新因子反馈给模型重训，模型效果反馈给因子搜索方向 |
| **架构** | Research阶段(假设→任务)→Development阶段(代码生成→执行)→Feedback阶段(评估→迭代)，Thompson Sampling调度（D6已实现） |

| **估时** | 7天 |
| **依赖** | G1完成后 |
